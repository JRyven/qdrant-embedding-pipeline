# Publication Checklist

Before publishing or sharing this repository externally:

- Remove local workspaces and editor files that contain private paths.
- Remove generated runtime data unless it is intentionally generic fixture data.
- Search for private hostnames, usernames, absolute local paths, internal project names, secrets, and personal notes.
- Keep example data neutral and instructional.
- Rewrite stale roadmap or planning material as public documentation, or remove it.
- If sensitive files were ever committed, remove them from Git history with a history-rewrite tool and rotate any exposed secrets.
